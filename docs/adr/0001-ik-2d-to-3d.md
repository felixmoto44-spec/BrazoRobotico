# Migración IK de 2D a 3D

El motor de cinemática inversa usaba originalmente vectores 2D (`angle_at_joint(x,y)`) para calcular la flexión de cada dedo. Tras investigar repos de GitHub de manos robóticas similares, migramos a vectores 3D (`angle_between_3d(x,y,z)`) que capturan profundidad — crítico para la abducción del pulgar. La versión 2D sobrevive como fósil en `_archive/v1-browser/`.

**Alternativas rechazadas:**
- **2D puro**: más rápido pero no captura la abducción del pulgar (el 90% de los repos usan comparación de eje X, no ángulos 2D)
- **Modelo biomecánico completo**: overkill para 5 servos, requeriría calibración por sujeto

**Parámetros recalibrados:** EMA α=0.25, dead zone 8°/12°, rate limit 5°/frame, stability 3 frames, SENSITIVITY=0.85, calibración per-finger.
